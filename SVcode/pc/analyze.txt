The simulation results show that the Program Counter module operates correctly. Whenever
reset = 1, the PC is synchronously reset to 0 at the next positive clock edge. Otherwise,
the PC is updated with the value of PCNext on each positive clock edge, which matches the
expected behaviour of the design.

At the beginning of simulation, the PC output appears as X. This is normal, because the
register has not yet been initialised before the first active clock edge. After the first
clock edge, the PC takes a definite value determined by either reset or PCNext, so the
module then behaves as expected.
